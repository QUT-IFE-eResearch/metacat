containerNodeId

displayOutsideMaxExtent

Object displayOutsideMaxExtent
    Should layers also be rendered outside the map extent? Default is false.

imageBuffer

Object imageBuffer
    for untiled wms layers: how many times should the map image be larger than the visible map. Large values mean slow loading, small ones mean many reloads when panning. Recommended values: 1-3, default is 2.

imageReproject

Object imageReproject
    For WMS on top of Google Maps you need to reproject the WMS image. This will stretch the WMS images to fit the odd sized google tiles. Default is false

loadingLayers

Object loadingLayers
    Number of layers that are currently being loaded

singleTile

Object singleTile
    For WMS layers: should they be regarded as normal WMS layers, with a single tile for the entire boundingbox or as a tiled WMS layer

tileBuffer

Object tileBuffer
    For tiled wms layers: how many rows of tiles should be preloaded outside the visible map? Large values mean slow loading, small ones mean longer delays when panning. Recommended values: 1-3, default is 2.

tileGutter

Object tileGutter
    For tiled wms layers: Overlap of map tiles in pixels. Useful for preventing rendering artefacts at tile edges. Recommended values: 0-15, default is 0 (no gutter at all).

tileSize

Object tileSize
    For tiled wms layers: how many pixels should the size of one tile be? Default is 256.

void refreshWmsLayers(objRef)
    Called after a feature has been added to a WFS. This function triggers the WMS basemaps to be redrawn. A timestamp param is added to the URL to ensure the basemap image is not cached. This function is triggered by the refreshWmsLayers event. If this event is fired with a as param, only that layer will be refreshed.

      objRef - reference to this widget
